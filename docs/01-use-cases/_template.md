# Use Case Template

## Goal

State the user outcome this use case achieves.

## Actors

List the primary actor and any supporting system actors.

## Preconditions

List what must already be true before the flow can begin.

## Trigger

Describe the event that starts the flow.

## Main Flow

1. Describe the happy path in short, numbered steps.

## Alternate Flows

1. List important variations such as loading, empty results, or validation branches.

## Postconditions

Describe what is true after the use case finishes.

## Business Rules Touched

Reference the relevant rules from `../00-overview/business-rules.md`.
